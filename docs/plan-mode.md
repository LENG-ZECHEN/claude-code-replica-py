## Overview

Plan Mode replicates Claude Code's read-only exploration phase as a
`PermissionMode(StrEnum)` carried on `AgentLoop`. The key invariant is
"constraint without cache breakage": the API `tools` field is
byte-identical across `NORMAL` and `PLAN`, so the prompt cache PREFIX
is preserved across every mode transition. Enforcement is achieved by
two cooperating mechanisms — (1) a per-turn `<system-reminder>`
USER-role attachment carrying `ENTER_PLAN_MODE_TEACHING_TEXT` teaches
the model to self-restrict (the load-bearing layer, ~95% of policing),
and (2) a runtime soft-deny in `AgentLoop._execute_one` returns
`is_error=True` for any tool whose `Tool.read_only` is False when the
loop is in `PLAN`. The soft-deny `ToolResult` content tells the model
exactly how to recover: `"Use exit_plan_mode to submit your plan for
approval, or use /plan to exit plan mode manually."`. Two model-driven
entry/exit tools (`enter_plan_mode`, `exit_plan_mode`) plus a
bidirectional `/plan` slash command in the REPL round out the surface;
the slash toggle is silent and preserves transcript history, while
`exit_plan_mode` blocks on `_confirm_exit_plan` user approval and
raises `PlanRejectedError` on rejection (which `ToolExecutor` converts
to `is_error=True`, bumping `plan_mode_exits_rejected`).

## Public API

- Enum + text: `PermissionMode.NORMAL` / `PermissionMode.PLAN`,
  `ENTER_PLAN_MODE_TEACHING_TEXT`, `PlanModeAttachment`.
- Tools: `enter_plan_mode`, `exit_plan_mode` (both `read_only=True`).
- Messages: `MessageType.ATTACHMENT_PLAN_MODE` +
  `Message.attachment_plan_mode(content)`.
- Tool field: `Tool.read_only: bool = False`.
- AgentLoop internals: `_permission_mode` field,
  `_set_permission_mode(mode, *, source="tool")`,
  `_maybe_arm_plan_mode_attachment()`.
- ContextBuilder: `build(plan_mode_attachment: PlanModeAttachment | None = None)`.
- Metrics: `plan_mode_entries`, `plan_mode_exits_approved`,
  `plan_mode_exits_rejected`, `plan_mode_write_attempts`, plus the
  computed `plan_mode_exits` property.
- Trace: `tracer.emit("permission", mode=..., source=...)` channel.
- REPL: `/plan` slash command (silent bidirectional toggle) plus the
  `_confirm_exit_plan` approval gate that fronts `exit_plan_mode`.
- Soft-deny error: `"Plan mode active: '<tool>' is not allowed. Only
  read-only tools may be called in plan mode. Use exit_plan_mode to
  submit your plan for approval, or use /plan to exit plan mode
  manually."`

- Reads from: `AgentLoop._permission_mode`, `Tool.read_only`.
- Writes to: `AgentLoop._permission_mode` via `_set_permission_mode`;
  `MessageType.ATTACHMENT_PLAN_MODE` injected by `ContextBuilder.build()`;
  the four `MetricsCollector` plan-mode counters.
- Depends on: `tools.py` (Tool.read_only field), `models.py`

  (permission channel).
