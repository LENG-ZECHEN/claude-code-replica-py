### Implementation notes from PLAN

This milestone is **tool layer only** — it does NOT add the teaching prompt
to the system prompt (that is M3). The model doesn't know this tool exists yet;
this milestone just makes the tool available for registration.

**`write_memory_entry` tool function:**
```python
def write_memory_entry(
    type: str,           # must be one of {user, feedback, project, reference}
    id: str,             # safe pattern: alphanumeric + _ + - + / (for subdirs)
    name: str,           # short slug, used as display name in MEMORY.md
    description: str,    # ≤150 chars
    body: str,           # memory content (checked against secret patterns)
    tags: list[str] | None = None,
) -> str:               # "Saved memory '<id>' as <type> (<abs_path>)"
```

**Validation (fail with `is_error=True`, informative message):**
- `type` not in `{"user", "feedback", "project", "reference"}` →
  `"Invalid type '<type>'. Must be one of: user, feedback, project, reference"`
- `id` not matching `_SAFE_ENTRY_ID_PATTERN` (imported from `memory.py`) →
  `"Invalid id '<id>'. Use only alphanumeric characters, hyphens, underscores, and forward slashes."`
- `len(description) > 150` →
  `"Description too long ({len(description)} chars). Maximum is 150 characters."`
- `_check_body_for_secrets(body)` returns a hit →
  `"Body contains what appears to be a secret (<pattern>). Refusing to save."`
- `_memory_writes_this_turn >= 3` (quota) →
  `"memory write quota exhausted this turn (max 3)"`

**Registration pattern** — mirror `register_snip_history_tool` from
`snip_tool_model.py` exactly:
```python
def register_write_memory_entry_tool(
    registry: ToolRegistry,
    project_memory: ProjectMemory,
) -> Tool:
    ...
```
Called from `AgentLoop._register_tools()` ONLY when `self._project_memory is not None`.

**AgentLoop changes:**
1. Add `project_memory: ProjectMemory | None = None` kwarg to `__init__`.
2. Store as `self._project_memory = project_memory`.
3. Add `self._memory_writes_this_turn: int = 0`.
4. Reset `_memory_writes_this_turn` to 0 at the **start** of `run()` and
   `run_stream()` (before the turn loop, not at the end).
5. Increment `_memory_writes_this_turn` inside the tool's closure, **before**
   the quota check (so the check is "if count >= 3, refuse").
6. In `_register_tools()`, call `register_write_memory_entry_tool(...)` when
   `self._project_memory is not None`.

**Upsert semantics**: calling `write_memory_entry` with the same `id` as an
existing memory overwrites it (new frontmatter + body). This is `ProjectMemory.save()`'s
existing behavior — the tool is a thin wrapper.
