```python
def find_relevant_memories(
    query: str,
    dir: Path,
    selector: Provider,
    *,
    already_surfaced: set[str],       # paths already shown this session
    read_file_state: set[str],        # paths the agent has already Read
    recent_tools: list[str],          # from collect_recent_successful_tools
    session_bytes_used: int,          # cumulative bytes injected this session
) -> list[MemoryHeader]:
```
4-gate short-circuit (ALL must pass; return [] if any fails):
1. `auto_memory_enabled` — (passed in as a gate kwarg OR read from the selector;
   check whether auto-memory is enabled; details depend on AgentLoop's state
   field; simplest: add a bool `auto_memory_enabled: bool = True` gate kwarg)
2. `query` is not empty (`query.strip() != ""`)
3. `query` is multi-word (`len(query.split()) > 1`)
4. `session_bytes_used < 60 * 1024`  (60KB ceiling)

On passing all gates:
- Call `selector.call_selector(system=SELECT_MEMORIES_SYSTEM_PROMPT, user=query_payload, ...)`
  where `query_payload` includes: query, the memory manifest (from `format_memory_manifest`),
  and optionally `recent_tools`.
- On `SelectorError`: fall back to existing Jaccard `MemorySelector`.
- Validate returned filenames against `scan_memory_files(dir)` manifest —
  any filename not in the manifest is silently dropped (hallucination guard).
- Filter out paths in `already_surfaced` and `read_file_state`.
- Return up to 5 `MemoryHeader` entries.
- Emit on Tracer `memory_select` channel:
  `{selected_count, manifest_size, session_bytes_used, fallback_used}`.
