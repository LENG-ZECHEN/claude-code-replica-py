### Implementation notes from PLAN

This is a **pure infrastructure milestone** — no AgentLoop change, no loop

**Provider Protocol addition:**
```python
class Provider(Protocol):
    def call(self, ...) -> ProviderResponse: ...
    def stream_call(self, ...) -> ...: ...
    def call_selector(
        self,
        *,
        system: str,
        user: str,
        output_schema: dict,
        max_tokens: int = 256,
    ) -> dict: ...  # parsed JSON matching output_schema
```

**`SelectorError` exception:**
```python
class SelectorError(Exception):
    """Raised by call_selector on API failure, malformed JSON, or schema mismatch."""
```

**`MockProvider.call_selector`:**
- Constructor gains `selector_responses: list[dict] = field(default_factory=list)`.
- `call_selector()` pops from the front of `selector_responses` (round-robin via
  a separate `_selector_idx` counter) or raises `SelectorError` when the list is
  exhausted. The existing `responses` list for regular `call()` is NOT changed.

**`OpenAIProvider.call_selector`:**
- Constructor gains `selector_model: str = "gpt-4o-mini"`.
- Calls OpenAI (or DashScope) completions API with:
  - `model=self.selector_model`
  - `response_format={"type": "json_object"}`
  - `temperature=0`
  - `max_tokens=max_tokens`
  - `messages=[{"role":"system","content":system},{"role":"user","content":user}]`
- On success: `json.loads(response.choices[0].message.content)`.
- On API error: raise `SelectorError(str(e))`.
- On `json.JSONDecodeError`: raise `SelectorError("malformed JSON: ...")`.
- On schema mismatch (returned dict missing a required key from `output_schema`):
  raise `SelectorError("schema mismatch: missing key ...")`.
