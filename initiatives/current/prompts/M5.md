### Implementation notes from PLAN

**New state fields on AgentLoop:**
```python
self._extract_memories_enabled: bool = False  # set from kwarg / flag
self._last_memory_message_uuid: str | None = None  # at-least-once cursor
self._extraction_in_progress: bool = False    # re-entrancy guard
self._turns_since_last_extraction: int = 0    # throttle counter
```

**`_run_stop_hooks(result: LoopResult) -> LoopResult`:**
- Called in `run()` and `run_stream()` BEFORE returning.
- Currently just calls `_maybe_extract_memories(result)`.
- Returns the (possibly mutated) result.

**`_maybe_extract_memories(result: LoopResult) -> None`:**
7-layer gating in order (short-circuit on first False):
1. `self._is_subloop` → skip (extractor running inside extractor)
2. `self._extract_memories_enabled` → skip (user hasn't opted in)
3. `self._auto_memory_enabled` → skip (memory feature disabled)
4. `self._extraction_in_progress` → skip (re-entrancy guard)
5. `hasMemoryWritesSince(messages, self._last_memory_message_uuid)` →
   skip if agent already wrote memories this turn (no need to extract)
6. throttle: `self._turns_since_last_extraction < throttle_n` → skip
7. run: set `_extraction_in_progress=True`, call `ExtractMemoriesRunner(...).run(...)`,
   advance `_last_memory_message_uuid` to the last message uuid in the transcript
   (ONLY on success), reset `_turns_since_last_extraction=0`, increment metrics.
   On exception: do NOT advance cursor (at-least-once), set `_extraction_in_progress=False`.

**`hasMemoryWritesSince(messages, since_uuid) -> bool`:**
Module-level function (not a method). Scans `messages` from the end until
finding a message with `uuid == since_uuid` (or reaching the start). Returns
True if any `tool_use` block with `name == "write_memory_entry"` is found in
an assistant message after the cursor position.

**`_last_memory_message_uuid` cursor semantics:**
- Initialized to `None` (meaning "scan from the beginning").
- After a successful extraction run: set to the uuid of the LAST message in
  `self._messages` at the time of the call.
- After a FAILED extraction run (exception raised): cursor is NOT advanced.
  Next turn will retry from the same position (at-least-once).

**Flag precedence** (mirror `cli._resolve_threshold`):
- `--extract-memories` CLI flag → `True` if present
- `SIMPLE_AGENT_EXTRACT_MEMORIES=1` env var → `True` if set
- Default: `False`
Do NOT add `--extract-memories` to `--aggressive-thresholds` — extraction
has cost implications and must be an explicit opt-in.

**Throttle:**
- Default throttle_n = 1 (extract every turn).
- `SIMPLE_AGENT_EXTRACT_THROTTLE=N` env var → extract every N turns.
- `_turns_since_last_extraction` increments each turn in the stop hook.
  Reset to 0 after a successful extraction.

**MetricsCollector additions in `metrics.py`:**
- `extract_invocations: int = 0` — incremented each time the runner actually
  runs (all 7 gates pass).
- `extract_writes: int = 0` — incremented by `len(result.written_paths)` on
  success.
- `/stats` output line: add `extract_invocations=N extract_writes=M`.

**is_subloop guard in ExtractMemoriesRunner:**
- M4 constructed ExtractMemoriesRunner as a pure engine.
- M5 must ensure that when `_maybe_extract_memories` sets
  `_extraction_in_progress=True` and calls `runner.run()`, the runner's
  inner `provider.call()` loop cannot recursively trigger another extraction.
- The simplest implementation: the ExtractMemoriesRunner's inner loop uses
  `provider.call()` directly (not via an AgentLoop), so there is no stop hook
  path. Verify this is the case. If M4's implementation happened to use an
  inner AgentLoop, refactor it in M5 to use direct provider.call().
