### Implementation notes from PLAN

This is a **pure data-layer change** — no tool registration, no loop wiring,
no CLI flag other than the `migrate-format` subcommand.

**Mini YAML parser (hand-rolled, no PyYAML):**
- Only needs to parse 4-5 frontmatter keys: `name`, `type`, `description`, `created_at`, optionally `tags`.
- Parser reads lines between the opening and closing `---` delimiters.
- Fail-soft: if frontmatter is malformed (broken YAML, missing `---` closer,
  fields wrong type), yield `MemoryHeader(id=..., description=None, name=..., type=None)`
  — the file is still usable by ID. Never raise.
- `FRONTMATTER_MAX_LINES = 30`: if no closing `---` appears within 30 lines
  of the opening `---`, treat as no frontmatter (description=None).

**New `MemoryHeader` dataclass** (replaces the old dict-based approach):
```python
@dataclass(frozen=True)
class MemoryHeader:
    id: str                     # relative path without .md suffix, e.g. "user/role"
    name: str                   # from frontmatter `name:` or "" if missing
    type: str | None            # "user" | "feedback" | "project" | "reference" | None
    description: str | None     # one-line summary, or None if parse failed
    path: Path                  # absolute path to .md file
    mtime: float                # from os.stat
```

**`scan_memory_files(dir: Path) -> list[MemoryHeader]`:**
- Recursively `glob("**/*.md")` under `dir`.
- Exclude files named `MEMORY.md` (basename check).
- For each file, read first 30 lines to parse frontmatter → MemoryHeader.
- Return list sorted by mtime descending (newest first).
- Path traversal defense: skip any resolved path not under `dir` (use
  `Path.is_relative_to(dir)` — Python 3.9+).

**`ProjectMemory.save(entry)`** now writes `<id>.md` with:
```
---
name: {entry.name}
type: {entry.type}
description: {entry.description}
created_at: {entry.created_at.isoformat() if set else datetime.now().isoformat()}
---

{entry.body}
```
Subdir support: extend `_SAFE_ENTRY_ID_PATTERN` to allow `/`-separated
path segments (e.g., `user/role`, `feedback/testing`). Traversal defense via
`Path.is_relative_to(root)` before writing.

**`_update_manifest(dir)` → atomic write:**
- Collects all `MemoryHeader[]` from `scan_memory_files(dir)`.
- Builds lines: `- [{h.name or h.id}]({rel_path}) — {h.description or "(no description)"}`.
- Applies 200-line truncation first; if the file at 200 lines exceeds 25KB,
  also truncate to fit 25KB. In both cases, append a warning footer:
  `> [truncated — {N} entries omitted to stay within limits]`.
- Uses `tempfile.NamedTemporaryFile` + `os.replace` (atomic) — mirror the
  pattern in `transcript._atomic_write_json`.

**`ProjectMemory.all()`** reads both `.md` and legacy `.json` during the
compat window:
1. `scan_memory_files(dir)` → all `.md` entries.
2. Also glob `*.json` (excluding any that have a `.md` counterpart at the
   same id) → load via existing JSON path.
3. Return merged list.

**`simple-agent memory migrate-format` subcommand:**
- In `memory_cli.py`, add a `migrate-format` subcommand under the existing
  `memory` group.
- For each `<id>.json` in the memory directory, if `<id>.md` does NOT yet
  exist: read JSON, write `.md` with frontmatter, leave the `.json` in place
  (idempotent — running twice is safe).
- Print: `Migrated N entries. Run again to verify (idempotent).`
