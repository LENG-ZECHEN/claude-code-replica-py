### Implementation notes from PLAN

Two additive CLI flags. **No new abstractions.**

**`--microcompact-minutes N`** (both `cli.py` and `openai_cli.py`):

- Plumbs to `MicroCompactor(threshold_minutes=N)` via the existing
  three-state precedence: explicit flag > `_AGGRESSIVE_THRESHOLDS`
  preset > built-in default of 60.
- Pattern: see how `--microcompact-keep-recent` is wired through
  `cli._resolve_threshold` in `cli.py`; mirror for `openai_cli` if not
  already there.
- The aggressive preset already sets `microcompact_minutes=1`. The
  new flag exposes the same lever directly.
- **`N=0` must work**: it means "any age qualifies" so microcompact
  fires on the next turn after a compactable tool_result.

  **`N=0` vs the existing `MicroCompactor(threshold_minutes<1)`
  guard.** `observable-thresholds-harden M2` added a guard test
  that rejects values `< 1`. Check that test first. If `0` trips it:
  either relax the guard to `N < 0` (and update its tests), OR set
  the new flag's accepted minimum to `1` AND note in your HANDOFF
  Section 5 that **M2 scenario 03 must adjust** (use
  `--microcompact-minutes 1` plus a brief sleep instead of the
  `--microcompact-minutes 0` originally planned).

**`--max-turns N`** (`openai_cli.py` REPL only):

- REPL exits cleanly after `N` user turns, taking the same shutdown
  path as `/exit` (dump SessionMemory, save session if one was loaded,
  exit 0).
- Counter checked at the top of the read loop in
  `_drive_repl_session` (or the openai_cli equivalent) before
  `input()` blocks.
- Purpose: deterministic finish so M2 can capture artifacts without
  manual `/exit`.

**Tests (at least 3 new cases):**

- `tests/test_cli.py`: `--microcompact-minutes` parsed and propagates

- `tests/test_openai_cli.py`: `--microcompact-minutes` parsed and

- `tests/test_openai_cli.py` (or `test_openai_cli_repl.py`):
