Faithful behaviors to replicate (do NOT simplify these away):
  - **The lock file is BOTH the mutex AND the timing state.** Its mtime
    *is* lastConsolidatedAt; its body *is* the holder PID. There is NO
    separate state file. `read_last_consolidated_at()` is a single
    `stat()` returning the mtime in ms (or `0` if the file is absent).
  - **Staleness vs liveness.** A lock younger than `HOLDER_STALE_MS`
    (1h) held by a *live* PID blocks acquire (`return None`). A lock
    older than 1h, OR held by a dead/unparseable PID, is **reclaimable**
    even within 1h (consolidationLock.ts:60-68). Replicate the
    PID-reuse guard: past 1h, reclaim unconditionally regardless of
    liveness.
  - **acquire returns the PRE-acquire mtime** so the caller can hand it
    to `rollback_consolidation_lock(prior_mtime)` after a failed dream.
    On a fresh acquire (no prior file) it returns `0`.
  - **rollback rewinds the time gate.** After a failed fork, rollback
    clears the PID body (so the still-running process does not look like
    a holder) and `os.utime`s the file back to `prior_mtime`, so the
    time gate re-opens on the next eligible turn instead of waiting a
    full 24h. `prior_mtime == 0` → unlink (restore no-file).
  - **Scan throttle is closure/instance-scoped, not on disk.** In TS,
    `lastSessionScanAt` lives inside the `initAutoDream()` closure
    (autoDream.ts:123), reset per test. The replica has no async startup
    closure; carry this as **instance state on a small gate object** (or
    an explicit `last_scan_at_ms` argument injected by the caller) so it
    is testable without a module global. Pick one and document it in the
    module header.
  - **Current-session exclusion.** `list_sessions_touched_since(since_ms)`
    returns the session ids touched after `since_ms`; the SESSION-GATE
    step excludes the current session id before comparing against
    `MIN_SESSIONS` (autoDream.ts:163-166). Make the exclusion explicit
    and tested (a single recently-touched current session must NOT
    count toward the ≥5).
